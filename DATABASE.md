# "Investment Brain" Database Schema Design

This document defines the relational architecture for the `Market-Analysis` persistent storage. The goal is to move from volatile JSON caching to a robust, queryable Hybrid Storage system (SQLite + File System).

---

## 🏗 Schema Overview

The database uses **SQLite** for structured financial data and the **Local File System** for unstructured investor documents (PDFs).

### 1. `assets` (Metadata Table)
Stores the master list of tickers and their static identification data.
| Column | Type | Description |
| :--- | :--- | :--- |
| `symbol` | TEXT (PK) | Ticker symbol (e.g., AAPL). |
| `name` | TEXT | Full company or fund name. |
| `asset_type` | TEXT | STOCK, ETF, or INDEX. |
| `sector` | TEXT | Economic sector (Stocks only). |
| `industry` | TEXT | Industry group (Stocks only). |
| `exchange` | TEXT | Listing exchange (e.g., NASDAQ). |
| `currency` | TEXT | Reporting currency (e.g., USD). |
| `last_updated` | DATETIME | Last time the full `info` was refreshed. |

### 2. `indices` (Index Metadata)
Stores the master list of indices and ETFs.
| Column | Type | Description |
| :--- | :--- | :--- |
| `symbol` | TEXT (PK) | Index/ETF identifier (e.g., SP500, QQQ). |
| `name` | TEXT | Full name (e.g., S&P 500). |
| `is_etf` | BOOLEAN | True if it's a tradeable ETF, False if a theoretical index. |
| `last_updated` | DATETIME | Last time membership was refreshed. |

### 3. `index_constituents` (Membership Table)
Maps the many-to-many relationship between indices and assets.
| Column | Type | Description |
| :--- | :--- | :--- |
| `index_symbol` | TEXT (FK) | Link to `indices.symbol`. |
| `asset_symbol` | TEXT (FK) | Link to `assets.symbol`. |
| `weight` | REAL | Optional: Percentage weight in the index. |

### 4. `financial_statements` (Time-Series Data)
Stores raw line items from Income Statements, Balance Sheets, and Cash Flow statements.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | INTEGER (PK) | Auto-incrementing ID. |
| `symbol` | TEXT (FK) | Link to `assets.symbol`. |
| `statement_type` | TEXT | INCOME, BALANCE, or CASHFLOW. |
| `period_type` | TEXT | ANNUAL or QUARTERLY. |
| `fiscal_date` | DATE | The date the report covers. |
| `metric_key` | TEXT | The standardized line item key (e.g., `NetIncome`). |
| `value` | REAL | The numeric value. |

### 5. `analysis_snapshots` (Results History)
Stores the scores generated by the tool, allowing for historical comparison of a stock's quality over time.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | INTEGER (PK) | Auto-incrementing ID. |
| `symbol` | TEXT (FK) | Link to `assets.symbol`. |
| `timestamp` | DATETIME | When the analysis was performed. |
| `profile` | TEXT | The investor profile used (e.g., 'value'). |
| `total_score` | REAL | The final percentage score (0-100). |
| `results_json` | TEXT | A JSON-string of individual metric results for breakdown. |
| `benchmark_version`| TEXT | The version of benchmarks used (e.g., '1.0.0'). |

### 6. `document_index` (Qualitative Vault)
Indexes the local PDF vault for investor relations documents.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | INTEGER (PK) | Auto-incrementing ID. |
| `symbol` | TEXT (FK) | Link to `assets.symbol`. |
| `doc_type` | TEXT | 10K, 10Q, PRESENTATION, or TRANSCRIPT. |
| `fiscal_year` | INTEGER | The year the document covers. |
| `fiscal_period` | TEXT | FY, Q1, Q2, Q3, or Q4. |
| `file_path` | TEXT | Relative path to the file in the PDF Vault. |
| `file_hash` | TEXT | SHA-256 hash to prevent duplicates and detect corruption. |
| `download_date` | DATETIME | When the document was added to the vault. |

### 7. `sector_benchmarks` (Dynamic Peer Targets)
Stores sector-specific overrides for metric evaluation.
| Column | Type | Description |
| :--- | :--- | :--- |
| `sector` | TEXT (PK) | Economic sector name. |
| `metric_key` | TEXT (PK) | The metric this benchmark applies to. |
| `version` | TEXT (PK) | The benchmark version (e.g., '1.0.0'). |
| `benchmark_type` | TEXT | 'best_worst' or 'target_width'. |
| `value_a` | REAL | 'best' or 'target' value. |
| `value_b` | REAL | 'worst' or 'width' value. |
| `last_updated` | DATETIME | 30-day freshness policy. |

### 8. `metrics_history` (Time-Series Metrics)
Stores historical snapshots of individual metrics for trend analysis.
| Column | Type | Description |
| :--- | :--- | :--- |
| `symbol` | TEXT | Ticker symbol. |
| `metric_key` | TEXT | e.g., 'pe_ratio'. |
| `value` | REAL | Numeric value. |
| `timestamp` | DATETIME | Time of recording. |

### 9. `investor_profiles` (Strategy Metadata)
Stores the master list of available investment strategies.
| Column | Type | Description |
| :--- | :--- | :--- |
| `name` | TEXT (PK) | Unique name of the profile (e.g., 'growth'). |
| `description` | TEXT | Human-readable description. |

### 10. `profile_weights` (Strategy Logic)
Stores the weights assigned to each metric within a specific profile.
| Column | Type | Description |
| :--- | :--- | :--- |
| `profile_name` | TEXT (FK) | Link to `investor_profiles.name`. |
| `metric_key` | TEXT | The metric being weighted. |
| `weight` | REAL | Numerical weight multiplier. |

### 11. `global_benchmarks` (Core Evaluation Rules)
Stores the master scoring rules for each asset type.
| Column | Type | Description |
| :--- | :--- | :--- |
| `asset_type` | TEXT (PK) | 'STOCK' or 'ETF'. |
| `metric_key` | TEXT (PK) | e.g., 'pe_ratio'. |
| `version` | TEXT (PK) | The benchmark version (e.g., '1.0.0'). |
| `name` | TEXT | Human-readable display name. |
| `formula_type`| TEXT | 'sigmoid', 'bell_curve', 'linear', 'threshold'. |
| `unit` | TEXT | 'multiplier', 'percentage', 'currency', or NULL. |
| `is_decimal` | BOOLEAN| True if 0.05 means 5% in raw data. |
| `display_key` | TEXT | Optional secondary key for labels. |
| `params_json` | TEXT | JSON string of scorer params (e.g. `{"best": 15, "worst": 50}`). |
| `weight` | REAL | Default multiplier for the metric. |

## 🛠 Strategic Implementation Plan

### Step 1: Foundation (Infrastructure)
Create `core/database/` with a `DatabaseManager` using `sqlite3`. Implement schema migrations to ensure tables are created correctly.

### Step 2: Index & Asset Persistence
Modify `get_index_components` to store fetched lists in the `indices` and `index_constituents` tables (7-day cache).

### Step 3: Sector Intelligence
Implement the `sector_benchmarks` table. Populate it initially from `sectors.json`, with a logic to auto-refresh benchmarks every 30 days based on aggregated DB data.

### Step 4: JSON Migration
Complete the migration of `investor_profiles.json`, `sectors.json`, `stock.json`, and `etf.json` into relational tables. **Delete legacy JSON files.**

### Step 5: Passive Snapshotting
Update the `Orchestrator` to record every analysis run in `analysis_snapshots` and `metrics_history`. This builds a historical record of "Quality Over Time."

### Step 6: Read-Through Caching
Update `YFinanceProvider` to query the database first. Only fetch from yfinance if data is missing or stale (> 24 hours).

### Step 7: Qualitative Integration
Implement the PDF downloader and `document_index` to support qualitative business analysis.

## 🔄 Database Maintenance & Syncing

To maintain ease of use while leveraging the power of a database, the following tools are planned for implementation:

### 1. YAML Configuration Sync (`make db-sync-configs`)
Allows for human-friendly editing of `global_benchmarks` and `investor_profiles` in YAML files, which are then bi-directionally synced with the database. This keeps strategies in Version Control (Git) while they live in the DB.

### 2. Bulk Sector Editing (`make db-export-sectors` / `make db-import-sectors`)
Enables exporting `sector_benchmarks` to CSV for mass-editing in Excel or Numbers, and re-importing the refined values back into the "Investment Brain."

### 3. Interactive CLI Editor (`make db-edit`)
A lightweight interactive prompt for making surgical adjustments to individual weights or benchmark thresholds without writing SQL.
